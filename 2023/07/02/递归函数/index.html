递归（Recursive）

递归，在数学与计算机科学中，是指在函数的定义中使用函数自身的方法。递归一词还较常用于描述以自相似方法重
复事物的过程。例如，当两面镜子相互之间近似平行时，镜中嵌套的图像是以无限递归的形式出现的。也可以理解为
自我复制的过程。递归的定义如下：

  • 在数学和计算机科学中，递归指由一种（或多种）简单的基本情况定义的一类对象或方法，并规定其他所
    有情况都能被还原为其基本情况。

递归函数（recursive function）或方法是指自己调用自己的一类过程，分为直接递归和间接递归。在直接递归中,
递归函数f的代码包含了调用f的语句；而在间接递归中，递归函数f调用了函数g，g又调用了函数h，如此下去，直
到又调用了f。下面先回顾两个数学概念——数学函数的递归定义和归纳证明。

递归的数学函数

数学中经常有这样的函数，它自己定义自己。例如：n的阶乘函数f(n)=n!，n为整数:
$$ f(n)=\begin{cases}
1 & n \leq 1 \\
nf(n-1) & n > 1
\tag{1-1}
\end{cases}
$$
当n小于或等于1时，f(n)的值为1，例如f(-3)=f(0)=f(1)=1。当n大于1时，f(n)是递归定义的，因为右侧也有f。但
这不会导致循环定义，因为右侧f的参数小于左侧f的参数，例如，f(2)=2f(1)，因为f(1)=1,所以f(2)=2*1=2，以此
类推f(3)=3f(2)=3*2=6。
假定f(n)是直接递归的。要使函数f(n)的递归定义有一个完全的形式，就需要满足如下的条件：

  • 有一个基础部分，它包含n的一个或多个值，对这些值，f(n)是直接定义的（即不用递归就能求解）。为
    简单起见，假定f的定义域为非负证书，基础部分包含$0\leq n \leq k$，其中k为非负常熟。（$n\geq k$的情形
    也是可能的，但很少见。
  • 在递归部分，右侧f有一个小于n的参数，因此重复应用递归部分可以把右侧f的表达式变为基础部分。

在公式（1-1）中，基础部分是f(n)=1，$n \leq 1$；递归部分是f(n)=nf(n-1),右侧f的参数是n-1，比n小。重复
应用递归部分将f(n-1)转变为f(n-2),f(n-3)，…..，直到f(1),而f(1)属于基础部分，例如：
$$f(5)=5f(4)=20f(3)=60f(2)=120f(1)$$
注意，递归部分的每一次应用都使的更趋近于基础部分，直到最后应用基础部分，得到f(5)=120。从这个例子中可
以看到的是：$f(n)=n(n-1)(n-2)…1 (n \geq 1)$。
递归定义的另一个例子就是斐波那契额数列：
$$ F_0=0, F_1=1, F_n=F_{n-1} + F_{n-2} \qquad(n > 1) \tag{1-2} $$
其中$F_0=1$和$F_1=1$是基础部分，$F_n = F_{n-1} + F_{n-2}$是递归部分。右侧的函数参数比n小。要使公式
（1-2）成为完全递归形式，从n > 1开始反复应用递归部分，每次n的值都要减去1或2，最终将右侧F的表达式转为
基础部分的表达式。例如：$F_4 = F_3+F_2 = F_2+F_1+F_1+F_0 = 3F_1+2F_0 = 3$。

归纳

与递归函数有关的第二个概念——归纳证明。
在一个归纳证明中，如果要证明下列公式成立：
$$\sum_{i=0}^n i=n(n+1)/2 \qquad n\geq 0 \tag{1-3}$$
证明方法是，首先检验，对n的一个或多个基础值（一般n=0就可以），公式成立；其次，假设当n从0到m时公式是成立
的，其中m是任意一个大于或者等于最大基础值的整数；最后，根据上述的假设证明，当n等于m+1时公式也成立。这种
证明方法有三个部分，分别是：

  • 归纳基础（induction base）
  • 归纳假设（induction hypothesis）
  • 归纳步骤 （induction step）

下面根据上述三段论方法开始证明公式（1-3）。

归纳基础

当n=0时，很显然公式是成立的，等式两边很容易求得都为0。

归纳假设

假设当$n \leq m$时公式成立，其中m是任意大于或等于0的整数（假设n=m时公式也可以成立）。

归纳步骤

在此阶段需要证明当n=m+1时公式成立，从归纳假设可知：$\sum_{i=0}^mi=m(m+1)/2$成立，因此：
$$\sum_{i=0}^{m+1}i=m+1+\sum_{i=0}^m i=m+1+m(m+1)/2=(m+1)(m+2)/2$$
即当n=m+1时公式亦成立，所以公式（1-3）成立。
归纳证明看似是一个循环证明——因为在第二步时假设的是一个正确的结论。其实并非如此，就像递归定义并不是
循环定义一样。每一个正确的归纳证明都有一个归纳基础部分，它与递归定义的基础部分相似，而在归纳步骤时使
用的是在归纳基础部分已经检验的正确结果，反复应用归纳步骤，把证明部分转为基础部分所具有的形式，最后就
得以得到最终结果。

递归函数

正确的递归函数必须包含基础部分。每一次递归调用，其参数值都比上一次的参数值要小，从而重复调用递归函数
使其参数值到达基础部分的值。

排列

从n个不同元素的所有排列中确定一个最佳排列。例如：a、b和c的排列有abc、acb、bac、bca、cba和cab。n个元
素的排列个数是n的阶乘n！。如果要输出这n个元素的排列，使用非递归的方式去做将是很复杂且比较困难的，但
是如果编写递归函数则会变的相对容易简单许多。下面来分析拆解一下该过程的样子。
设E={$e_1,e_2,….e_n$}是n个元素的集合，求E的元素的所有排列。
令$E_i$表示从E中去除第i个元素$e_i$以后的集合，令perm(X)表示集合X的元素所组成的所有排列，令$e_i$.perm(X)
表示在perm(X)中的每个排列加上前缀$e_i$之后的派列表。例如，E={a,b,c},$E_1$={b,c},perm($E_1$)=(bc,cb),
$e_1.perm(E_1)$=(abc, acb)。
当n=1时，是递归的基础部分。这时的集合E只有一个元素e，因此只有一个排列：perm(E)=(e)。当n > 1时，perm(E)
是一个表：$e_1.perm(E_1),e_2.perm(E_2),e_3.perm(E_3),….,e_n.perm(E_n)$。这个定义是用n个集合perm(X)来
定义集合perm(E),其中每个X包含n-1个元素，它成为递归步骤。这是一个既有基础部分，又有递归部分的完整的递归
定义。
根据上述的递归定义
当n=3且E = {a,b,c}时，有perm(E)=a.perm({b,c}),b.perm({a,c}),c.perm({b,a})。
同理，perm({b,c})=b.perm({c}),c.perm({b})。
因此，a.perm({b,c})=ab.perm({c}),ac.perm({b})=ab.c,ac.b=(abc,acb)
同理，perm({a,c})=a.perm({c}),c.perm({a})。
因此，b.perm({a,c})=ba.perm({c}),bc.perm({a})=ba.c,bc.a=(bac,bca)
同理，perm({b,a})=b.perm({a}),a.perm({b})。
因此，c.perm({b,a})=cb.perm({a}),ca.perm({b})=cb.a,ca.b=(cba,cab)
所以，perm(E)=(abc,acb,bac,bca,cba,cab)。
a.perm({b,c})代表了两个排列：abc和acb，其中a是它们的前缀，perm({b,c})是它们的后缀，同样ac.perm({b})表

子集

使用递归输出一个有n个元素的集合的所有子集。 

#include <iostream>
#include <vector>
#include <string>

template<typename T>
void generateSubsets(const std::vector<T>& nums, std::vector<T>& current, int index) {
    // 输出当前子集
    for (const T& num : current) {
        std::cout << num << " ";
    }
    std::cout << std::endl;

    // 递归终止条件
    if (index == nums.size()) {
        return;
    }

    // 递归生成子集
    for (int i = index; i < nums.size(); ++i) {
        current.push_back(nums[i]);
        generateSubsets(nums, current, i + 1);
        current.pop_back();
    }
}

template<typename T>
void printSubsets(const std::vector<T>& nums) {
    std::vector<T> current;
    generateSubsets(nums, current, 0);
}

int main() {
    int n;
    std::cout << "集合中元素总个数: ";
    std::cin >> n;

    std::vector<std::string> nums(n);
    for (int i = 0; i < n; ++i) {
        std::cout << "请输入元素 " << i + 1 << ": ";
        std::cin >> nums[i];
    }

    std::cout << "子集有: " << std::endl;
    printSubsets(nums);

    return 0;
}
